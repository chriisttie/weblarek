# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Данные

В этом разделе описаны интерфейсы и типы, которые определяют структуру данных, используемых в приложении.
Интерфейсы обеспечивают типизацию данных и гарантируют, что все части приложения работают с данными в едином формате.

#### Интерфейс товара IProduct

Описывает структуру данных одного товара в каталоге.
interface IProduct {
id: string; // Уникальный идентификатор товара
description: string; // Полное описание товара
image: string; // URL изображения товара
title: string; // Название товара
category: string; // Категория товара
price: number | null; // Цена товара (может быть null, если цена не указана)
}
Назначение: Используется для хранения и передачи данных о товаре между сервером, моделью каталога и компонентами интерфейса.

#### Интерфейс покупателя IBuyer

Описывает данные, которые покупатель указывает при оформлении заказа.
interface IBuyer {
payment: TPayment; // Выбранный способ оплаты
email: string; // Адрес электронной почты
phone: string; // Номер телефона
address: string; // Адрес доставки
}
Назначение: Используется моделью покупателя для хранения данных заказа и валидации перед отправкой на сервер.

#### Тип способа оплаты TPayment

Определяет допустимые значения для поля оплаты.
type TPayment = 'card' | 'cash';
Назначение: Ограничивает возможные значения поля payment только двумя вариантами: оплата картой или онлайн-платёж.

#### Тип ошибок валидации IBuyerErrors

Описывает формат объекта, который возвращает метод валидации покупателя.
`type IBuyerErrors = Partial<Record<keyof IBuyer, string>>;`
Назначение: Используется для типизации объекта с ошибками валидации, где ключ - имя поля, значение - текст ошибки. Если поле валидно, оно отсутствует в объекте.

#### Тип заказа IOrder

Описывает данные, отправляемые на сервер при оформлении заказа. Наследует поля из IBuyer.
interface IOrder extends IBuyer {
total: number; // Общая стоимость заказа
items: string[]; // Массив ID товаров в заказе
}
Назначение: Используется для передачи данных заказа в метод createOrder класса ApiLarek.

#### Tип ответа сервера IOrderResponse

Описывает ответ сервера после успешного создания заказа.
interface IOrderResponse {
id: string; // Уникальный идентификатор заказа
total: number; // Сумма заказа
}

#### Тип ответа сервера IProductsResponse

Описывает ответ сервера на запрос списка товаров.
interface IProductsResponse {
total: number; // Общее количество товаров
items: IProduct[]; // Массив товаров
}

### Модели данных

В этом разделе описаны классы, которые реализуют работу с данными приложения. Каждый класс отвечает за свой участок данных и инкапсулирует логику работы с ними.

#### Класс Catalog

Класс отвечает за хранение списка всех доступных товаров и управление товаром, выбранным для подробного просмотра. Не отвечает за отображение — только за хранение и предоставление данных.
Конструктор:
`constructor(items: IProduct[] = [])` - принимает массив товаров для инициализации каталога. По умолчанию массив пустой.
Поля класса:
`items: IProduct[]` - хранит массив всех товаров, доступных в каталоге.
`previewProduct: IProduct | null` - хранит товар, выбранный для отображения в модальном окне предпросмотра.
Методы класса:
`setItems(items: IProduct[]): void` - сохраняет переданный массив товаров в каталог.
`getItems(): IProduct[]` - возвращает массив всех товаров из каталога.
`getItemById(id: string): IProduct | undefined` - возвращает товар по его уникальному идентификатору.
`setPreviewProduct(product: IProduct): void` - сохраняет товар для подробного отображения.
`getPreviewProduct(): IProduct | null` - возвращает текущий товар для предпросмотра.

#### Класс Cart

Класс отвечает за хранение списка товаров, выбранных пользователем для покупки. Предоставляет методы для управления содержимым корзины и расчёта итоговой стоимости.
Конструктор:
`constructor()` - конструктор не принимает параметров. Корзина инициализируется пустой.
Поля класса:
`items: IProduct[]` - хранит массив товаров, добавленных в корзину.
Методы класса:
`getItems(): IProduct[]` - возвращает массив товаров в корзине.
`add(product: IProduct): void` - добавляет товар в корзину.
`remove(productId: string): void` - удаляет товар из корзины по его ID.
`clear(): void` - полностью очищает корзину.
`getTotalPrice(): number` - возвращает общую стоимость всех товаров в корзине.
`getCount(): number` - возвращает количество товаров в корзине.
`has(productId: string): boolean` - проверяет наличие товара с указанным ID в корзине.

#### Класс Customer

Класс отвечает за хранение контактных данных покупателя и данных о способе оплаты. Реализует валидацию данных перед отправкой заказа.
Конструктор:
`constructor()` - конструктор не принимает параметров. Данные инициализируются пустыми значениями.
Поля класса:
`payment: TPayment | null` — выбранный способ оплаты (изначально null).
`email: string` - адрес электронной почты покупателя.
`phone: string` - номер телефона покупателя.
`address: string` - адрес доставки заказа.
Методы класса:
`set(data: Partial<IBuyer>): void` - устанавливает переданные поля. Позволяет обновлять данные частично, не затрагивая остальные поля.
`get(): IBuyer` - возвращает объект со всеми данными покупателя.
`clear(): void` - сбрасывает все поля к пустым значениям.
`validate(): IBuyerErrors` — проверяет валидность всех полей. Возвращает объект с ошибками, где ключ — имя поля, значение — текст ошибки. Для строковых полей (email, phone, address) перед проверкой применяется .trim(), чтобы игнорировать пробелы. Если поле валидно, оно отсутствует в возвращаемом объекте.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Слой коммуникации

В этом разделе описан класс, который отвечает за взаимодействие с сервером API. Класс использует композицию с базовым классом Api для выполнения HTTP-запросов.

#### Класс ApiLarek

Назначение и зона ответственности:

Класс отвечает за получение данных с сервера (список товаров) и отправку данных о заказе. Не хранит и не преобразует данные - только передаёт их между сервером и другими слоями приложения.
Конструктор:
`constructor(api: IApi)` — принимает объект с базовыми методами для HTTP-запросов (get, post).
Поля класса:
`api: IApi` — ссылка на объект API для выполнения запросов.
Методы класса:
`getProducts(): Promise<IProductsResponse>` — выполняет GET-запрос на /product/, возвращает полный объект ответа сервера.
`createOrder(order: IOrder): Promise<IOrderResponse>` — выполняет POST-запрос на /order/, передаёт данные заказа и возвращает подтверждение.
Метод getProducts возвращает данные в том же виде, в каком их отдал сервер (IProductsResponse), без извлечения массива items.

## Слой представления (View)

В этом разделе описаны классы, отвечающие за отображение данных и обработку пользовательских действий. Все классы представления наследуются от базового класса Component.

#### Класс Card<T>

Базовый класс для всех карточек товаров. Содержит общую логику отображения заголовка, цены, категории и кнопки действия.

Конструктор:
constructor(container: HTMLElement) - принимает корневой DOM-элемент карточки.

Поля класса:
`titleElement: HTMLElement` - элемент для отображения заголовка товара.
`priceElement: HTMLElement` - элемент для отображения цены.
`categoryElement: HTMLElement | null` - элемент для отображения категории (может отсутствовать в некоторых карточках).
`button: HTMLButtonElement | null`- кнопка действия (может отсутствовать).

Методы класса:
`set title(value: string)` - устанавливает текст заголовка.
`set price(value: number | null)` - устанавливает цену товара. Если цена null, отображается текст "Бесценно".
`set category(value: string)` - устанавливает категорию товара. Применяет соответствующий модификатор класса из categoryMap и отображает человекочитаемое название из categoryNames.
`set buttonState(isDisabled: boolean)` - блокирует или разблокирует кнопку действия.

#### Класс CardWithImage<T>

Промежуточный класс для карточек с изображением и категорией. Наследуется от Card<T>.

Конструктор:
`constructor(container: HTMLElement)` - принимает корневой DOM-элемент карточки.

Поля класса:
`protected readonly image: HTMLImageElement` - элемент изображения товара.
`protected readonly categoryElement: HTMLElement | null` - элемент для отображения категории.

Методы класса:
`protected setProductImage(imagePath: string, alt: string)` - устанавливает src и alt изображения.
`protected setProductCategory(value: string)` - устанавливает категорию товара с применением модификатора из categoryMap.

#### Класс CardCatalog

Карточка товара в каталоге на главной странице. Наследуется от Card<IProduct>.

Конструктор:
`constructor(events: IEvents, container: HTMLElement)` - принимает брокер событий и корневой элемент.

Поля класса:
`image: HTMLImageElement` - элемент изображения товара.

Методы класса:
`set product(data: IProduct)` - устанавливает все данные товара: заголовок, цену, категорию, изображение. Также устанавливает data-id атрибут для идентификации товара.

События:
`product:select` - генерируется при клике на карточку (не на кнопку). Передаёт productId для открытия модального окна с подробной информацией.
`product:add` - генерируется при клике на кнопку "Купить". Передаёт productId для добавления товара в корзину.

#### Класс CardCart

Карточка товара в корзине. Наследуется от Card<IProduct & { id: string }>.

Конструктор:
`constructor(events: IEvents, container: HTMLElement)` - принимает брокер событий и корневой элемент.

Поля класса:
`image: HTMLImageElement | null` - элемент изображения (может отсутствовать).
`removeButton: HTMLElement` - кнопка удаления товара из корзины.

Методы класса:
`set product(data: IProduct & { id: string })` - устанавливает данные товара для отображения в корзине.

События:
`cart:item:remove` - генерируется при клике на кнопку удаления. Передаёт productId для удаления товара из корзины.

#### Класс CardProductFull

Полная карточка товара в модальном окне предпросмотра. Наследуется от CardWithImage<IProduct>.

Конструктор:
`constructor(container: HTMLElement, onAction?: () => void)` - принимает корневой элемент и колбэк для действия с корзиной.

Поля класса:
`protected readonly description: HTMLElement` - элемент для отображения полного описания товара.
`private isButtonDisabledForPrice: boolean` - флаг блокировки кнопки из-за отсутствия цены.

Методы класса:
`set product(data: IProduct)` - устанавливает все данные товара включая описание.
`setButtonText(text: string)` - обновляет текст кнопки, если она не заблокирована из-за цены.

Особенности:

- Клик по кнопке вызывает колбэк onAction без передачи данных.
- Текст кнопки определяется в презентере на основе состояния корзины.
- Нет геттеров для получения данных (согласно требованиям).

#### Класс Form<T>

Базовый класс для всех форм. Содержит общую логику работы с кнопкой отправки и отображением ошибок.

Конструктор:
`constructor(container: HTMLElement)` - принимает корневой элемент формы.

Поля класса:
`submitButton: HTMLButtonElement` - кнопка отправки формы.
`errorsContainer: HTMLElement` - контейнер для отображения сообщений об ошибках.

Методы класса:
`set submitButtonState(isDisabled: boolean)` - блокирует или разблокирует кнопку отправки.
`clearErrors()` - очищает все сообщения об ошибках.

#### Класс FormPayment

Форма выбора способа оплаты и ввода адреса доставки на первом шаге оформления заказа.

Конструктор:
`constructor(events: IEvents, container: HTMLElement)` - принимает брокер событий и корневой элемент формы.

Поля класса:
`paymentButtons: NodeListOf<HTMLButtonElement>` - коллекция кнопок выбора способа оплаты.
`addressInput: HTMLInputElement` - поле ввода адреса доставки.

Методы класса:
`set payment(data: Partial<IBuyer>)` - устанавливает данные оплаты и адреса.
`set errors(errors: IBuyerErrors)` - отображает сообщения об ошибках валидации.

События:
`form:payment:change` - генерируется при выборе способа оплаты.

## Презентер (main.ts)

Презентер содержит основную логику приложения и отвечает за связь между моделями и представлениями. Весь код презентера находится в файле src/main.ts.

### Инициализация

При загрузке приложения создаются экземпляры всех необходимых классов:
- Модели: Catalog, Cart, Customer, ApiLarek (создаются однократно)
- Представления: Header, Gallery, Modal, Basket, FormPayment, FormContacts (создаются однократно)
- Карточки: CardCatalog, CardCart, ProductFull (пересоздаются при изменении данных)

### Обработчики событий

Презентер подписывается на все события, генерируемые моделями и представлениями:

#### События от моделей:

`catalog:change` - при изменении каталога товаров. Создает карточки для каждого товара через CardCatalog и передает их в галерею.
`product:preview` - при выборе товара для предпросмотра. Создает полную карточку через ProductFull, отображает её в модальном окне и сохраняет ссылку для последующего обновления кнопки.
`cart:change` - при изменении корзины. Обновляет счётчик в шапке, обновляет текст кнопки в активном модальном окне, перерисовывает каталог и корзину если она открыта.
`customer:change` - при изменении данных покупателя. Используется для финальной валидации перед отправкой заказа.

#### События от представлений:

`product:select` - при клике на карточку в каталоге. Запрашивает товар из модели и генерирует событие product:preview.
`product:add` - при клике на кнопку "Купить/Удалить". Проверяет наличие товара в корзине и добавляет или удаляет его. Закрывает модальное окно после действия.
`cart:item:remove` - при клике на кнопку удаления в корзине. Удаляет товар из корзины.
`basket:open` - при клике на иконку корзины. Формирует содержимое корзины и открывает модальное окно.
`form:payment:submit` - при успешном заполнении формы оплаты. Сохраняет данные в модели покупателя и открывает форму контактов.
`form:contacts:submit` - при успешном заполнении формы контактов. Выполняет валидацию, формирует заказ и отправляет его на сервер через ApiLarek. При успехе отображает окно подтверждения, очищает корзину и данные покупателя.
`modal:close` - при закрытии модального окна. Сбрасывает ссылки на активные формы и карточки.
`order:success:close` - при закрытии окна успеха. Закрывает модальное окно.

### Загрузка данных

При инициализации приложения выполняется запрос к API для получения списка товаров через метод getProducts() класса ApiLarek. При успешном ответе товары сохраняются в модели Catalog, что автоматически генерирует событие catalog:change и запускает отрисовку каталога.

#### Утилиты и константы

Файл src/utils/constants.ts
Содержит константы, используемые во всём приложении:
`API_URL` - базовый адрес API сервера. Формируется из переменной окружения VITE_API_ORIGIN.
`CDN_URL` - базовый адрес для изображений товаров. Используется для формирования полных URL картинок.
`categoryMap` - объект соответствия категорий товара их CSS-модификаторам. Используется для применения цветного фона к элементу категории.
`categoryNames` - объект человекочитаемых названий категорий. Используется для отображения текста категории пользователю.

Файл src/utils/utils.ts
Содержит вспомогательные функции:
`ensureElement<T>(selector, context?)` - находит элемент по селектору или возвращает переданный элемент. Выбрасывает ошибку если элемент не найден.
`ensureAllElements<T>(selectorElement, context?)` - находит все элементы по селектору или возвращает переданную коллекцию.
`cloneTemplate<T>(query)` - клонирует содержимое HTML template элемента.
`bem(block, element?, modifier?)` - утилита для генерации имён классов по методологии БЭМ.
`createElement<T>(tagName, props?, children?)` - фабрика для создания DOM-элементов с настройками.

### События приложения

#### События от моделей

`catalog:change - данные: { items: IProduct[] }`. Генерируется при изменении списка товаров в каталоге.
`product:preview - данные: { product: IProduct }`. Генерируется при установке товара для предпросмотра.
`cart:change` - данные: нет. Генерируется при любом изменении содержимого корзины.
`customer:change` - данные: нет. Генерируется при изменении данных покупателя.

#### События от представлений

`product:select` - данные: { productId: string }. Генерируется при клике на карточку товара в каталоге.
`product:add` - данные: { productId: string }. Генерируется при клике на кнопку "Купить/Удалить".
`cart:item:remove` - данные: { productId: string }. Генерируется при клике на кнопку удаления товара из корзины.
`basket:open` - данные: нет. Генерируется при клике на иконку корзины в шапке.
`form:payment:change` - данные: { payment: TPayment }. Генерируется при выборе способа оплаты.
`form:payment:submit` - данные: { payment: TPayment, address: string }. Генерируется при успешной валидации формы оплаты.
`form:contacts:submit` - данные: Partial<IBuyer>. Генерируется при успешной валидации формы контактов.
`modal:close` - данные: нет. Генерируется при закрытии модального окна.
`order:success:close` - данные: нет. Генерируется при закрытии окна успешного заказа.
